# I18n Specification

## Overview
The `I18nService` provides a decentralized multi-language support system. Instead of one massive file, translations are split by language into modular files.

## Structure
- `I18nService.js`: Core management logic (loading, switching, param replacement).
- `translations/`: Directory containing language-specific files:
  - `en.js` (English - Master)
  - `es.js` (Spanish)
  - `ca.js` (Catalan)
  - `eu.js` (Basque/Euskera)
  - `gl.js` (Galician/Galego)

## Usage
### In Code
```javascript
import { i18n } from './shared/core/i18n/I18nService.js';

// Simple translation
const text = i18n.t('victory'); // "VICTORY!"

// With parameters
// en.js -> { greeting: "Hello, {name}!" }
const welcome = i18n.t('greeting', { name: 'Arthur' }); // "Hello, Arthur!"
```

## Adding a New Language
1.  Create `js/engine/shared/core/i18n/translations/[lang].js`.
2.  Export a constant with the same name as the file.
3.  Import it in `I18nService.js` and add it to the `this.translations` object.

## Best Practices
- **Keys**: Use snake_case for keys (e.g., `btn_start_game`).
- **Fallbacks**: If a key is missing in the current language, the system will fallback to English (`en`).
